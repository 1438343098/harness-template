# apps/ Directory Navigation — AGENTS.md

> All frontend application directories. Each subdirectory is an independent frontend project with its own tech stack and AGENTS.md.

---

## Directory Structure

```
apps/
├── AGENTS.md              # This file
├── web/                   # Example: main site (user-facing)
│   └── AGENTS.md
├── admin/                 # Example: admin dashboard
│   └── AGENTS.md
└── mobile/                # Example: mobile app (React Native / Flutter)
    └── AGENTS.md
```

> Actual subdirectories are created by Claude Code after `/process-requirements` resolves the projects.

---

## Sub-project Naming Convention

| Type | Recommended directory name | Example tech stack |
|------|-----------|-----------|
| User-facing Web | `web` | React + TypeScript |
| Admin dashboard | `admin` | React + Ant Design |
| Marketing site | `landing` | Next.js |
| Mobile app | `mobile` | React Native / Flutter |
| Desktop app | `desktop` | Electron + React |

---

## Each Sub-project Contains

- `AGENTS.md` — tech stack, directory structure, and code standards for that project
- Actual code (generated by Claude Code)
- `package.json` / corresponding language build config

---

## View All Frontend Project Statuses

```bash
cat features.json | python3 -c "import sys,json; d=json.load(sys.stdin); [print(p['id'],p['name'],p['tech_stack']) for p in d.get('projects',{}).get('apps',[])]"
```
