# services/ Directory Navigation — AGENTS.md

> All backend service directories. Each subdirectory is an independent backend service that can use different languages and frameworks.

---

## Directory Structure

```
services/
├── AGENTS.md              # This file
├── api/                   # Example: main API service (Node.js)
│   └── AGENTS.md
├── worker/                # Example: background tasks (Python)
│   └── AGENTS.md
└── gateway/               # Example: API gateway (Go)
    └── AGENTS.md
```

> Actual subdirectories are created by Claude Code after `/process-requirements` resolves the services.

---

## Sub-service Naming Convention

| Type | Recommended directory name | Example tech stack |
|------|-----------|-----------|
| Main API | `api` | Node.js + Express / Python + FastAPI |
| Auth service | `auth` | Node.js + JWT |
| Background tasks | `worker` | Python + Celery |
| Real-time communication | `realtime` | Node.js + Socket.io |
| File handling | `storage` | Go / Node.js |
| Message queue | `queue` | Python / Go |
| API gateway | `gateway` | Go / Node.js |

---

## Each Sub-service Contains

- `AGENTS.md` — language, framework, API spec, and startup instructions for that service
- Actual code (generated by Claude Code)
- `package.json` / `pyproject.toml` / `go.mod` and other build configs
- `.env.example` — environment variable template

---

## Cross-service Communication Convention

- **Synchronous calls:** REST API (`http://service-name:port/api/v1/...`)
- **Async messaging:** Message queue (defined in `features.json`'s `infrastructure` section)
- **Shared types:** Placed in the `shared/types/` directory at the project root

---

## View All Backend Service Statuses

```bash
cat features.json | python3 -c "import sys,json; d=json.load(sys.stdin); [print(p['id'],p['name'],p['tech_stack'],p['language']) for p in d.get('projects',{}).get('services',[])]"
```
